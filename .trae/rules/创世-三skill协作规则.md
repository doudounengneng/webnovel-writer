## 四、自动识别规则

根据用户输入类型，自动判断进入哪个阶段：

| 输入类型 | 自动进入 |
|---------|---------|
| 灵感碎片、想法、感觉 | 阶段一：灵魂激活 |
| 大纲、设定、架构 | 阶段二：骨架搭建 |
| 人设、人物关系 | 阶段三：灵魂灌注 |
| "写第一章"、"扩写" | 阶段四：正文输出 |
| "帮我看看"、"诊断" | 阶段五：灵魂校对 |

## 五、质量门（每个阶段输出前必过）

### 墨白质量门

- [ ] 这段话是否符合人物灵魂？
- [ ] 情感温度是否到位？
- [ ] 是否像真人说的话/做的事？
- [ ] 人物行为逻辑是否自洽？
- [ ] 是否避免了工具人风险？

### 创世神质量门

- [ ] 世界观逻辑是否闭环？
- [ ] 伏笔是否有对应回收节点？
- [ ] 爽点密度是否达标（章小爽/三章中爽/十章大爽）？
- [ ] 三线是否并行不冲突？
- [ ] 节奏是否紧凑无废话？
- [ ] 章末钩子是否埋下？

### 网文大师质量门

- [ ] 节奏是否紧凑？
- [ ] 爽点是否即时兑现？
- [ ] 章末钩子是否埋下？
- [ ] 格式是否符合标准？
- [ ] 错别字/语病是否修正？

## 六、关键节点暂停机制

| 阶段 | 是否暂停 | 暂停条件 |
|------|---------|---------|
| 阶段一：灵魂激活 | ✅ 必须暂停 | 每轮追问后等你回答 |
| 阶段二：骨架搭建 | ✅ 必须暂停 | 骨架提案完成后等你确认 |
| 阶段三：灵魂灌注 | ✅ 必须暂停 | 人设卡完成后等你确认 |
| 阶段四：正文输出 | ❌ 自动 | 先输出，有问题再回头 |
| 阶段五：灵魂校对 | ✅ 必须暂停 | 校对完成后等你验收 |
| 阶段六：定稿交付 | ✅ 必须暂停 | 最终确认 |

## 七、设定文档自动生成机制

每次协作完成后，系统自动生成以下文档并存入项目目录：

| 文档名 | 内容 |
|--------|------|
| **世界观设定.md** | 世界背景、修炼体系、势力格局、时间线 |
| **人物设定.md** | 主角灵魂卡、所有配角灵魂卡 |
| **金手指设定.md** | 系统机制、能力体系、升级逻辑 |
| **章节规划.md** | 卷/章大纲、爽点分布、伏笔清单 |
| **三线设计.md** | 主线明线、反派暗线、身份秘线 |
| **创作笔记.md** | 本次协作的关键决策、修改记录 |

文档路径：`d:\文本\编程\项目\{项目名}\设定文档\`

## 八、半自动协作原则

1. **保留碰撞**：关键节点必须等你确认，不自动跳过
2. **自动执行**：日常输出自动跑，不需要重复输入触发词
3. **自审先行**：每个Skill输出前必须先过质量门，再呈现给你
4. **设定留存**：所有确认的设定自动写入文档，供后续创作查阅

## 九、整合升级：十层宇宙 × 三Skill

如果你希望创作时更有"根系感"和"生长感"，可以在进入三Skill流程前，先用**十层宇宙**找到故事根系的裂缝。

### 两种启动方式

| 触发词 | 进入模式 |
|--------|---------|
| **创世** | 直接进入三Skill协作流程（阶段一~六） |
| **十层** | 先用十层宇宙找到裂缝，再进入完整七阶段流程 |

### 完整七阶段 vs 三Skill六阶段

完整流程（十层+三Skill）：
```
阶段0：灵感激活（十层宇宙）→ 阶段1~7（三Skill）
```

三Skill流程（直接）：
```
阶段一：灵魂激活 → 阶段二：骨架搭建 → ... → 阶段六：定稿交付
```

### 十层宇宙的核心价值

在正式构建世界观之前，先回答这个问题：
> **"为了让这个欲望成立，哪一层的规则必须被打破？"**

这个被打破的层，就是故事根系中的**裂缝**——从这里开始，世界才会一寸一寸地被照亮。

详见完整协议：**十层×三Skill协作规则.md**

## 十、规则文件定位

本规则文件是**通用协作协议**，不绑定任何具体项目。

每次新项目启动时：
- 输入"创世"进入三Skill协作流程
- 输入"十层"进入十层+三Skill完整流程
- 流程中确认的所有设定自动写入对应项目的设定文档
- 本规则文件保持通用，不记录具体项目数据
