# 阶段契约定义

> 参考vibewriting的contracts机制，为每个阶段定义显性的输出契约
>
> 📌 质量门规范源：[QUALITY_GATES.md](./QUALITY_GATES.md)；阶段定义规范源：[创世/SKILL.md](../skills/创世/SKILL.md)

---

## 一、契约概述

### 契约层级

| 层级 | 定义 | 约束力 |
|------|------|--------|
| **主契约** | 每个阶段必须产出的核心文件 | 强约束，必须满足才能进入下一阶段 |
| **次契约** | 每个阶段的可选/推荐产出 | 弱约束，建议满足 |
| **质量门** | 通过质量门的检查项 | 通过/不通过 |

> 📐 以下模板定义了各阶段产出的**文件格式**（"输出什么"）。每个阶段的**质量判断标准**（"做到什么程度算好"）详见 [QUALITY_GATES.md](./QUALITY_GATES.md)。两者配合使用：模板管格式，质量门管深度。

### 契约文件命名规范

```
阶段X-{阶段名}/
├── 主契约产出/
│   ├── 种子冲击报告.md      # 阶段0
│   ├── 灵魂激活卡.md        # 阶段1
│   ├── 骨架激活报告.md      # 阶段2
│   ├── 世界观设定.md        # 阶段3
│   ├── 人物灵魂卡.md        # 阶段4
│   └── 章节XX.md            # 阶段5
├── 次契约产出/
│   ├── 三线设计.md          # 阶段3
│   ├── 伏笔清单.md          # 阶段3
│   └── 校对报告.md          # 阶段6
└── 质量门检查表.md          # 各阶段通用
```

- 道：[根本规律]

## 三、契约验证流程

### 自动验证时机

1. **阶段切换时**：检查上一阶段的主契约是否满足
2. **暂停点确认前**：检查当前阶段的产出是否满足契约
3. **断点续跑时**：检查项目状态与实际文件的一致性

### 验证失败处理

```
【契约验证失败】

阶段：阶段3（骨架搭建）
主契约：世界观设定.md
状态：❌ 未满足

缺失项：
1. ❌ 修炼体系未完整（缺少突破条件）
2. ❌ 伏笔清单缺少长线伏笔

建议：
1. 请补充修炼体系的突破条件
2. 请添加至少2个长线伏笔

是否继续？ [继续修改/跳过进入下一阶段/查看详细]
```

## 四、契约与锚点的关系

| 契约类型 | 锚点关系 |
|---------|---------|
| 主契约 | 锚点的具体化/文档化 |
| 次契约 | 锚点的扩展/细化 |
| 质量门 | 锚点的验证机制 |

**锚点 → 契约的转化：**
```
种子核心欲望（锚点）→ 灵魂激活卡（契约）
五维命运标签（锚点）→ 骨架激活报告（契约）
三线架构（锚点）→ 三线设计（契约）
人物欲望（锚点）→ 人物灵魂卡（契约）
```
