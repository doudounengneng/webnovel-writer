# 断点续跑机制

> 参考vibewriting的checkpoint机制，为十层×三Skill体系建立完整的断点续跑系统

---

## 一、断点续跑核心概念

### 断点定义

| 断点类型 | 说明 | 保存内容 |
|---------|------|---------|
| **阶段断点** | 每个阶段完成时 | 阶段产出路径 + 锚点状态 |
| **章节断点** | 每章完成时 | 章节内容 + 伏笔状态 |
| **会话断点** | 每次会话结束时 | 全部状态 + 项目快照 |

### 状态文件

```
项目/
├── .trae/
│   ├── project-state.json       # 项目主状态
│   ├── checkpoint.json          # 断点记录
│   ├── anchors.json             # 锚点快照
│   └── session-log.json        # 会话日志
├── 设定文档/
│   └── 创作轨迹追踪.md         # 轨迹追踪文档
├── backup/
│   └── 2026-05-04_2030/       # 自动备份
└── ...
```

## 二、断点续跑命令

### 断点续跑命令列表

| 命令 | 别名 | 功能 |
|------|------|------|
| `继续创作` | `resume` | 从上次断点继续 |
| `继续写` | `continue` | 继续当前章节 |
| `恢复[项目名]` | `restore` | 恢复到指定项目 |
| `查看进度` | `status` | 查看当前进度 |
| `查看锚点` | `anchors` | 查看锚点摘要 |
| `清空断点` | `clear` | 清空所有断点，从头开始 |

### 命令处理流程

## 三、断点保存机制

### 自动保存触发时机

| 触发时机 | 保存内容 | 保存位置 |
|---------|---------|---------|
| 每个阶段完成时 | 阶段状态 + 产出路径 | checkpoint.json |
| 每章完成时 | 章节状态 + 伏笔状态 | checkpoint.json |
| 每个暂停点确认后 | 全部状态 | checkpoint.json |
| 会话结束时 | 全部状态 + 创建备份 | checkpoint.json + backup/ |
| 每30分钟 | 全部状态 | checkpoint.json |

### 保存流程

```
【自动保存触发】

[1] 保存类型：[阶段断点/章节断点/会话断点/定时断点]
[2] 收集状态数据...
    ├── project-state.json 更新
    ├── checkpoint.json 更新
    ├── anchors.json 更新
    └── session-log.json 追加

[3] 验证保存完整性...
    └── ✅ 验证通过

[4] 备份（如是会话断点）...
    ├── 创建目录：backup/2026-05-04_2030/
    ├── 复制所有设定文档
    ├── 复制所有已完成的章节
    └── 复制project-state.json

[5] 保存完成提示...
    ┌─────────────────────────────────────┐
    │ ✅ 自动保存完成                      │
    │ 时间：2026-05-04 20:30              │
    │ 类型：会话断点                       │
    │ 备份：backup/2026-05-04_2030/       │
    └─────────────────────────────────────┘
```

## 五、清空断点

### 清空命令

```
【清空断点】

⚠️ 警告：此操作将清空所有断点和备份

将要清空：
├── .trae/checkpoint.json
├── .trae/project-state.json
├── .trae/anchors.json
├── .trae/session-log.json
└── backup/（全部备份）

注意：设定文档和正文不会被删除

确认清空？ [输入"确认清空"以继续]
```

### 清空后状态

清空后，系统将：
1. 创建新的空白 `project-state.json`
2. 保留所有 `设定文档/` 下的文件
3. 保留所有 `正文/` 下的文件
4. 用户需要重新从阶段0开始，但可以复用现有设定文档

## 六、异常处理

### 断点文件损坏

```markdown
【错误：断点文件损坏】

检测到：.trae/checkpoint.json 文件损坏

可选操作：
A. 使用 backup/ 中最新的备份恢复
B. 查看可用的 backup/ 列表
C. 忽略错误，从 project-state.json 加载基础状态
D. 创建新的断点系统（会丢失进度）

请选择：[A/B/C/D]
```

### 设定文档缺失

```markdown
【警告：设定文档缺失】

检测到以下文档缺失：
├── 设定文档/人物灵魂卡.md
└── 设定文档/三线设计.md

当前阶段需要这些文档才能继续。

可选操作：
A. 从 backup/ 中恢复缺失的文档
B. 重新生成这些文档（会丢失部分人设细节）
C. 回退到阶段3，使用世界观设定.md重新生成

请选择：[A/B/C]
```

### 章节文件不一致

```markdown
【警告：章节文件不一致】

检测到：章节9的project-state.json记录与实际文件不符

project-state.json：第9章完成（100%）
实际文件：第9章草稿（50%，最后修改：3天前）

可选操作：
A. 以实际文件为准，更新project-state.json
B. 以project-state.json为准，恢复第9章备份
C. 重新开始第9章（保留大纲）

请选择：[A/B/C]
```

## 七、最佳实践

### 日常创作流程

```
1. 每次开始创作
   → 输入"继续创作"或"resume"

2. 系统自动恢复上次进度
   → 显示当前阶段、锚点摘要

3. 确认下一步
   → A. 继续当前阶段
   → B. 进入下一阶段
   → C. 查看锚点

4. 创作过程中
   → 系统每30分钟自动保存

5. 每次暂停/结束
   → 输入"保存"或直接结束
   → 系统自动创建会话断点 + 备份
```

### 断点续跑与备份关系

| 场景 | 触发 | 保存内容 |
|------|------|---------|
| 日常暂停 | 每30分钟/手动 | checkpoint.json |
| 会话结束 | 每次结束对话 | checkpoint.json + backup/ |
| 阶段完成 | 阶段质量门通过后 | checkpoint.json + 阶段报告 |
| 章节完成 | 章节质量门通过后 | checkpoint.json + 章节文件 |
| 紧急恢复 | 检测到异常时 | 立即保存当前状态 |

### 备份保留策略

```
backup/
├── 2026-05-04_2030/    # 最近会话备份
├── 2026-05-04_0830/    # 今日较早备份
├── 2026-05-03_2100/    # 昨日备份
├── 2026-05-01_1530/    # 上次创作备份
└── 2026-04-28_1000/    # 阶段切换备份
```

建议保留最近5个备份，旧备份自动清理。
