# 多Agent协作架构设计

> 参考vibewriting的Agent架构 + 创世神的5 Agent管线，为十层×三Skill体系设计完整的多Agent协作方案

---

## 一、多Agent架构概述

### 架构设计理念

```
┌─────────────────────────────────────────────────────────┐
│                    用户（Author）                        │
│                  输入灵感 / 确认方向                      │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                  主编Agent（Chief Editor）                 │
│              协调所有子Agent，统一输出风格                 │
│              - 意图解析                                    │
│              - 任务分发                                   │
│              - 结果整合                                   │
│              - 质量把控                                   │
└─────────────────────────┬───────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ 骨架Agent │   │ 灵魂Agent │   │ 肌肉Agent │
    │ (创世神)  │   │  (墨白)   │   │ (网文大师)│
    └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ 世界观   │   │ 人设     │   │ 正文     │
    │ 子Agent  │   │ 子Agent  │   │ 子Agent  │
    └──────────┘   └──────────┘   └──────────┘
```

### 三类Agent定位

| Agent类型 | 对应Skill | 核心职责 | 类比 |
|---------|----------|---------|------|
| **骨架Agent** | 创世神 | 世界观架构、伏笔布局、节奏规划 | 建筑师 |
| **灵魂Agent** | 墨白 | 人设灌注、情感把控、活人感 | 灵魂工程师 |
| **肌肉Agent** | 网文大师 | 正文输出、格式规范、错字校对 | 写手 |

## 二、主编Agent（Chief Editor）

### 主编Agent职责

```
主编Agent = 十层宇宙框架协调者 + 三Skill总调度
```

### 主编Agent的核心能力

**1. 意图解析**
```markdown
用户输入 → 主编Agent → 任务指令

用户："我想写一个穿越到洪荒世界的主角"
  ↓
意图解析：
  - 种子类型：角色种子
  - 触发Skill：十层宇宙（识别涉及的M层）
  - 触发阶段：阶段0（灵感激活）
  ↓
任务分发：启动阶段0工作流
```

**2. 阶段管理**
```markdown
主编Agent管理阶段切换：
  阶段0 → 阶段1 → 阶段2 → ... → 阶段7

每个阶段切换时：
  1. 检查上一阶段的契约是否满足
  2. 检查质量门是否通过
  3. 更新project-state.json
  4. 分发下一阶段任务
```

**3. 锚点传递**
```markdown
主编Agent维护锚点网络：

  阶段0锚点 → 阶段1锚点 → 阶段2锚点 → ... → 阶段7锚点

每个阶段结束时：
  1. 提取该阶段的锚点
  2. 验证锚点一致性
  3. 传递给下一阶段
  4. 更新创作轨迹追踪.md
```

**4. 冲突仲裁**
```markdown
当子Agent之间出现冲突时，主编Agent仲裁：

骨架Agent："世界观需要主角有血统限制"
灵魂Agent："血统限制会破坏主角的活人感"

主编Agent仲裁：
  1. 检查锚点：种子核心是否涉及血统？
  2. 检查阶段：当前是阶段3还是阶段4？
  3. 优先级：种子核心 > 世界观 > 活人感
  4. 裁决：保留血统限制（世界观），但降低其对活人感的影响
```

## 三、骨架Agent（创世神系）

### 骨架Agent职责

```
骨架Agent = 世界建筑师 + 伏笔布局者 + 节奏规划师
```

### 骨架Agent的子Agent矩阵

| 子Agent | 职责 | 输入 | 输出 |
|---------|------|------|------|
| **世界观Agent** | 构建世界底层逻辑 | M层五维 | 世界观设定.md |
| **伏笔Agent** | 埋设和回收伏笔 | 主线三线 | 伏笔清单.md |
| **节奏Agent** | 爽点分布、节奏把控 | 章节规划 | 章节节奏表 |
| **势力Agent** | 势力关系、冲突设计 | 世界观 | 势力格局图 |

### 世界观Agent详解

```markdown
【世界观Agent】

输入：
  - M层清单（L6/L7/L9）
  - 五维定义（道法术器势）
  - 命运标签（🧠👁️⚔️）

处理流程：
  1. 解析每个M层的"道"
     → 确定世界的根本规律

  2. 细化每个M层的"法"
     → 制定规则体系（修炼体系/社会制度/天道规则）

  3. 设计每个M层的"术"
     → 具体操作技巧（功法/技能/能力）

  4. 创造每个M层的"器"
     → 承载物件与场域（神器/秘境/势力）

  5. 分析每个M层的"势"
     → 矛盾方向与张力（冲突根源）

输出：
  - 世界观设定.md
  - 三界格局描述
  - 修炼体系完整定义
```

### 伏笔Agent详解

```markdown
【伏笔Agent】

输入：
  - 主线明线
  - 反派暗线
  - 身份秘线
  - 情节引爆点（⚔️标签）

处理流程：
  1. 从主线明线提取伏笔需求
     → 每个关键转折需要伏笔铺垫

  2. 从反派暗线提取伏笔需求
     → 反派行动需要线索暗示

  3. 从身份秘线提取伏笔需求
     → 身份秘密需要碎片铺垫

  4. 规划伏笔的埋设时机
     → 章级伏笔：提前1-3章
     → 卷级伏笔：提前5-10章
     → 书级伏笔：提前20+章

  5. 规划伏笔的回收时机
     → 必须有明确的回收节点
     → 回收时要有爆点

输出：
  - 伏笔清单.md
  - 伏笔埋设计划
  - 伏笔回收计划
```

### 节奏Agent详解

```markdown
【节奏Agent】

输入：
  - 章节规划
  - 爽点密度公式
  - 情节引爆点

处理流程：
  1. 确定全书爽点分布
     → 每章小爽点（打脸/装逼/暧昧）
     → 每3章中爽点（突破/逆袭/升温）
     → 每10章大爽点（名场面/高潮）

  2. 规划节奏曲线
     → 第1-3章：开篇钩子，节奏快
     → 第4-10章：设定展开，节奏稳
     → 第11-15章：第一高潮，节奏爆
     → 第16-20章：缓冲调整，节奏变
     → ...

  3. 设计章末钩子类型分布
     → 悬念型：20%
     → 冲突型：30%
     → 反转型：30%
     → 爆料型：20%

输出：
  - 章节节奏表
  - 爽点分布图
  - 钩子规划表
```

## 四、灵魂Agent（墨白系）

### 灵魂Agent职责

```
灵魂Agent = 人设工程师 + 情感调教师 + 活人感审核员
```

### 灵魂Agent的子Agent矩阵

| 子Agent | 职责 | 输入 | 输出 |
|---------|------|------|------|
| **人设Agent** | 主角/配角人设立体化 | 世界观设定 | 人物灵魂卡.md |
| **情感Agent** | 情感线规划、情绪把控 | 人设 | 情感线规划.md |
| **对话Agent** | 对话真实性、角色声音 | 人设 | 对话风格指南 |
| **活人感Agent** | 工具人检测、行为逻辑 | 章节草稿 | 活人感报告 |

### 人设Agent详解

```markdown
【人设Agent】

输入：
  - 世界观设定.md
  - 三线设计.md
  - 种子核心欲望

处理流程：
  1. 分析主角与世界观的契合度
     → 主角欲望如何在世界中实现？
     → 主角如何受到世界规则的制约？

  2. 主角立体化
     → 核心欲望（不能被优化的）
     → 性格标签（3-5个关键词）
     → 成长弧光（起点→终点）
     → 高光锚点（在哪一章必须高光）
     → 底线锚点（绝对不会做的事）

  3. 配角立体化
     → 每个配角独立动机
     → 与主角的关系
     → 在故事中的作用
     → 高光锚点

  4. 反派立体化
     → 独立权谋暗线
     → 反派自己的目标
     → 反派高光锚点

输出：
  - 人物灵魂卡.md（主角/配角/反派）
  - 人物关系图
```

### 情感Agent详解

```markdown
【情感Agent】

输入：
  - 人物灵魂卡.md
  - 章节规划

处理流程：
  1. 确定感情线类型
     → 单女主/多女主/无女主
     → 青梅竹马/天降系/竞争对手
     → 甜宠/虐恋/细水长流

  2. 规划情感升温节点
     → 第一次心动：第X章
     → 第一次危机：第X章
     → 表白/确认关系：第X章
     → 重大考验：第X章
     → 修成正果：第X章

  3. 设计情感节奏
     → 压抑 → 释放
     → 拉扯 → 升温
     → 危机 → 和解

  4. 规划情绪节点
     → 每3章一个小情绪点
     → 每10章一个大情绪爆点

输出：
  - 情感线规划.md
  - 情绪节奏表
```

### 活人感Agent详解

```markdown
【活人感Agent】

输入：
  - 章节草稿
  - 人物灵魂卡.md

处理流程：
  1. 工具人检测
     → 检查每个配角是否有独立动机
     → 检查配角行为是否服务于主角

  2. 对话真实性检验
     → 检查对话是否像真人
     → 检查不同角色的说话方式是否有区分
     → 检查是否有"完美台词"

  3. 行为逻辑检验
     → 检查主角行为是否符合人设
     → 检查配角行为是否在底线内
     → 检查是否有突然的行为转变

  4. 情感温度检验
     → 检查情感是否到位
     → 检查是否有情感起伏

输出：
  - 活人感报告.md
  - 修改建议清单
```

## 五、肌肉Agent（网文大师系）

### 肌肉Agent职责

```
肌肉Agent = 正文写手 + 格式规范师 + 错字校对员
```

### 肌肉Agent的子Agent矩阵

| 子Agent | 职责 | 输入 | 输出 |
|---------|------|------|------|
| **正文Agent** | 章节正文输出 | 章节规划+人设 | 章节草稿.md |
| **爽点Agent** | 爽点植入、即时兑现 | 章节草稿 | 爽点优化版 |
| **钩子Agent** | 章末钩子设计 | 章节草稿 | 钩子优化版 |
| **校对Agent** | 错字、语病、格式 | 定稿前章节 | 终稿 |

### 正文Agent详解

```markdown
【正文Agent】

输入：
  - 章节规划（本章任务）
  - 人物灵魂卡.md
  - 锚点网络（种子核心/三线/M层）
  - 上文上下文

处理流程：
  1. 解析本章任务
     → 本章核心剧情
     → 需要推进的伏笔
     → 需要触及的M层

  2. 锚点检查
     → 主角行为是否在锚点内？
     → 是否触及⚔️情节引爆点？
     → 是否需要铺垫/回收伏笔？

  3. 扩写正文
     → 叙事大气
     → 对话贴合人物
     → 节奏紧凑
     → 无废话

  4. 爽点植入
     → 核心爽点即时兑现
     → 辅助爽点合理分布

  5. 钩子设计
     → 悬念/冲突/反转/爆料
     → 为下章埋下期待

输出：
  - 章节草稿.md
  - 伏笔状态更新
```

### 校对Agent详解

```markdown
【校对Agent】

输入：
  - 章节草稿
  - 人物灵魂卡.md

处理流程：
  1. 错别字检查
     → 逐字扫描
     → 常见错字词库
     → 上下文推断

  2. 语病检查
     → 病句检测
     → 冗长句精简
     → 歧义句优化

  3. 格式检查
     → 段落划分
     → 对话格式
     → 专有名词统一
     → 数字使用规范

  4. 一致性检查
     → 与前文设定一致
     → 与锚点一致
     → 前后不矛盾

  5. 质量检查
     → 节奏是否紧凑
     → 爽点是否到位
     → 钩子是否清晰

输出：
  - 章节终稿.md
  - 校对报告.md
```

    ├── 阶段优先级：阶段3（世界观）> 阶段4（人设）

    A. 保留血统限制，但降低其对活人感的影响

    C. 暂时搁置，进入阶段4后再决定

    采纳方案A，并记录：
    ┌────────────────────────────────────┐

    │ 冲突点：血统限制 vs 活人感          │

    │ 后续跟进：阶段4时灵魂Agent需确认    │
    └────────────────────────────────────┘
```

## 八、Agent与Skill的对应关系

### Skill vs Agent映射

```
【十层宇宙 Skill】
    ↓
对应 Agent：
