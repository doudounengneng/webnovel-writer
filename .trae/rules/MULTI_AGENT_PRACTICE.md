## 三、智能体间协作方式

### 方式A：串行接力（推荐）

一个Agent完成后，将产出直接传递给下一个Agent：

```
【Agent-1 → CEO】你：创世 穿越洪荒世界

CEO启动项目：
  ↓ 调用 Agent-6（市场调研部）
  ↓ 调用 Agent-2（策划部）
  ↓ 调用 Agent-3（技术部）
  ↓ 调用 Agent-4（编辑部）
  ↓ ⚠️ 暂停（等你确认）
  ↓ 调用 Agent-5（生产部）
  ↓ 调用 Agent-4（编辑部-校对）
  ↓ 调用 Agent-7（质量优化部）
  ↓ 调用 Agent-5（生产部-定稿）
  ↓ 返回给你的最终成品
```

#### 串行接力指令示例

```
=== 第1步：CEO启动调研 ===
你：创世 穿越洪荒世界

[Agent-1 CEO] 收到指令，启动市场调研...

你（或CEO自动）：调研 洪荒爆款文

[Agent-6 市场调研部] 分析结果：
  爆款共性：金手指+扮猪吃虎+逆袭
  产出：调研报告.md

↓

=== 第2步：CEO交给策划部 ===

[Agent-1 CEO] 📋 传递调研报告给策划部

你（或CEO自动）：十层 基于调研，分析穿越洪荒的M层

[Agent-2 策划部] 分析结果：
  M层：L6心智意识 / L7社会文明
  产出：种子冲击报告.md

↓

=== 第3步：CEO交给技术部 ===

[Agent-1 CEO] 📋 传递M层数据给技术部

你（或CEO自动）：世界观 基于M层搭建洪荒世界

[Agent-3 技术部] 构建结果：
  世界观：洪荒+阐教+截教
  三线：已规划
  伏笔：8个
  产出：世界观设定.md

↓

...（以此类推，直到完成）
```

## 四、智能体启动配置

### 每个Agent的启动文件

为每个智能体创建一个独立的启动包，包含：

```
.trae/agents/
├── agent-1-ceo/
│   └── STARTUP.md    ← CEO启动指令
├── agent-2-planner/
│   └── STARTUP.md    ← 策划部启动指令
├── agent-3-architect/
│   └── STARTUP.md    ← 技术部启动指令
├── agent-4-editor/
│   └── STARTUP.md    ← 编辑部启动指令
├── agent-5-writer/
│   └── STARTUP.md    ← 生产部启动指令
├── agent-6-analyst/
│   └── STARTUP.md    ← 市场调研部启动指令
└── agent-7-optimizer/
    └── STARTUP.md    ← 质量优化部启动指令
```

### 启动文件模板

```markdown
# Agent-2 · 策划部启动指令

## 你的身份
你是创作公司的**策划部·世界观设计师**。

## 你的技能包
- 十层宇宙 SKILL.md

## 你的核心能力
- M层识别、五维定义
- 种子冲击报告生成
- 命运标签标记（🧠👁️⚔️）

## 你的输入格式
从CEO（Agent-1）接收：
- 项目需求
- 调研报告（如有）
- 前阶段锚点数据

## 你的输出格式
- 种子冲击报告.md
- 向下传递摘要（50字内）

## 协作协议
- 完成后通知CEO
- CEO决定是否进入下一阶段
- 接收CEO传递的下一阶段任务

---

## 启动命令

当收到CEO指令时，开始执行。
```

## 五、智能体间通信协议

### Handoff消息格式

每个Agent完成任务后，按以下格式传递给下一Agent：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【Handoff】Agent-X → Agent-Y

📋 传递摘要（50字内）：
[M层L6/L7已定义] + [种子核心：复仇者]

📦 产出文件：
[种子冲击报告.md]

📌 下一Agent任务：
[M层五维细化 + 世界观框架搭建]

✅ 质量门状态：
- M层≤3：✅
- 五维定义：✅
- 命运标签：✅

⚠️ 注意事项：
[无]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 中断与恢复

当某个Agent卡住时：
```
Agent-3（技术部）：世界观搭建遇到瓶颈，
不知如何处理修炼体系...

→ 返回给CEO Agent-1：

⚠️ 技术部卡点：修炼体系设计
需要：策划部补充M层五维细节
建议：回传策划部细化
```

## 七、实现方式说明

### 在Trae IDE中的实现

Trae IDE目前不支持同时运行多个独立智能体实例。因此有三种实现方式：

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **① 单Agent模拟多角色** | 当前对话作为CEO，模拟调用各部门 | ✅ 推荐，即开即用 |
| **② 多窗口协作** | 打开多个Trae窗口，各加载不同Skill | 团队分工专用 |
| **③ Task子智能体** | CEO用Task工具派发子任务 | 调研/分析类工作 |

### 方式①：单Agent模拟多角色（推荐）

当前对话就是CEO，用"角色切换"来模拟多Agent协作：

```
你：创世 穿越洪荒世界

CEO启动项目，依次模拟各部门：

[当前→策划部模式] 激活种子...（加载十层宇宙技能）
[当前→技术部模式] 搭建世界观...（加载创世神技能）
[当前→编辑部模式] 灌注人设...（加载墨白技能）
[当前→生产部模式] 输出正文...（加载网文大师技能）
[当前→编辑部模式] 校对...
[当前→质量优化部模式] 对比优化...
[当前→CEO模式] 汇总交付
```

**每个模式的切换，由CEO显式宣告**：

```
【切换至：策划部模式】
当前技能：十层宇宙
部门：策划部
任务：激活种子
开始执行...
```

## 九、最佳实践

1. **日常创作**：用方式①（单Agent模拟多角色），CEO一声令下全流程跑通
2. **深度调研**：用方式③（Task子智能体），派搜索Agent专项调研
3. **质量检查**：用方式①+③，CEO主流程+调子任务对比
4. **团队协作**：如有多个Trae用户，用方式②每人负责一个部门
