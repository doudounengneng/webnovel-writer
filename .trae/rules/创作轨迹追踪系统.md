```
## 断裂回溯索引

当用户在任何时候说"感觉不对"或"偏离了"时，快速回溯路径：

```
正文偏离 → 阶段5章节 → 阶段4人设 → 阶段1欲望 → 阶段0种子
     ↓           ↓           ↓           ↓
  锚点核对   人设弧光    规则打破    核心心跳
```

## 下次会话恢复提示

基于当前进度，下次会话开始时：

1. 读取 `project-state.json` 获取当前阶段和锚点摘要
2. 读取 `创作轨迹追踪.md` 获取最近锚点详情
3. 检查 `设定文档/` 目录下各文档的一致性
4. 按照当前阶段的产出要求继续工作
```

## 四、自动备份机制

### 备份触发时机

| 触发条件 | 备份内容 | 备份位置 |
|---------|---------|---------|
| 每日首次创作时 | 所有设定文档 + 轨迹追踪 | 项目/backup/日期时间/ |
| 每个暂停点确认后 | 当前阶段的产出文档 | 项目/backup/日期时间/ |
| 会话结束时 | 状态更新 + 轨迹追加 | 项目/backup/日期时间/ |

### 备份核查流程

每次开始创作前：
```
【备份核查】

检测到上次备份：[2026-05-04 20:30]
距今：8小时
备份状态：✅ 完整

最近修改的文档：
  - 创作轨迹追踪.md（1小时前）
  - 人物灵魂卡.md（2小时前）

是否需要立即备份？ [Y/N]
```

## 五、快速恢复命令

| 命令 | 功能 |
|------|------|
| `继续创作` | 按project-state.json恢复上次进度 |
| `查看锚点` | 显示当前所有锚点摘要 |
| `查看阶段X` | 显示指定阶段的产出和要求 |
| `检查一致性` | 检查各设定文档之间的矛盾 |
| `备份` | 手动触发立即备份 |
| `回溯[阶段]` | 回溯到指定阶段的锚点 |
