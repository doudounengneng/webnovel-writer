## 二、各部门职责

### CEO / 总编

| 属性 | 内容 |
|------|------|
| **技能** | 创世 |
| **部门** | 总裁办 |
| **工作地点** | 顶层（无需切换） |
| **核心职能** | 项目启动、工作流调度、结果汇总、最终交付 |

**日常操作**：
- 你给CEO下达命令，CEO自动调度各部门
- CEO不直接创作，它是协调者

## 三、公司级协作流程

### 日常立项流程

```
你：创世 穿越洪荒世界

CEO【创世】：
[1] 收到指令：穿越洪荒世界
[2] 启动市场调研 → 市场调研部【小说拆文】
   → 分析同类爆款洪荒文的特点
   → 产出：调研报告
[3] 启动策划 → 策划部【十层宇宙】
   → 基于调研报告激活种子
   → 产出：种子冲击报告
[4] 启动技术 → 技术部【创世神】
   → 基于策划方案搭建世界观
   → 产出：世界观设定.md
[5] 启动编辑 → 编辑部【墨白】
   → 基于世界观灌注人设
   → 产出：人物灵魂卡.md
[6] 启动生产 → 生产部【网文大师】
   → 基于章节规划输出正文
   → 产出：章节正文.md
[7] 编辑审核 → 编辑部【墨白】
   → 校对情感/活人感
   → 产出：校对报告.md
[8] 质量对比 → 质量优化部【网文对比】
   → 与大神作品对比
   → 产出：优化建议
[9] 生产交付 → 生产部【网文大师】
   → 定稿交付
   → 产出：章节-定稿.md
[10] 运营发布 → 运营部【运营大师】
    → 按最佳时间发布
    → 产出：运营日报
[11] 数据追踪 → 数据分析部【数据分析师】
    → 追读率/转化率分析
    → 产出：数据分析报告
[12] 营销推广 → 内容营销部【营销策划师】
    → 推广文案/拉票
    → 产出：推广方案
[13] 完本后 → IP开发部【IP策划师】
    → IP价值评估/改编规划
    → 产出：IP评估报告
```

### 跨部门协作矩阵

| 场景 | 主导部门 | 协作部门 | 协作内容 |
|------|---------|---------|---------|
| 项目启动 | CEO | 市场调研部 | 先调研再立项 |
| 种子激活 | 策划部 | 市场调研部 | 调研数据支撑 |
| 世界观搭建 | 技术部 | 策划部 | M层数据移交 |
| 人设灌注 | 编辑部 | 技术部 | 世界观数据输入 |
| 正文输出 | 生产部 | 技术部+编辑部 | 章节规划+人设 |
| 灵魂校对 | 编辑部 | 质量优化部 | 校对+对比 |
| 质量优化 | 质量优化部 | 生产部 | 优化后反馈 |
| **每日发布** | **运营部** | **生产部** | **定时发布章节** |
| **数据复盘** | **数据分析部** | **运营部** | **数据驱动优化** |
| **新书营销** | **内容营销部** | **策划部+运营部** | **书名/简介/推广** |
| **IP开发** | **IP开发部** | **市场调研部** | **完本后评估** |

## 四、公司运营命令

### 日常命令

| 输入 | 相当于 | CEO执行 |
|------|--------|---------|
| `创世 [灵感]` | 立项启动 | 召集策划部+技术部+生产部 |
| `调研 [类型]` | 市场调研 | 派市场调研部拆文 |
| `对比 [章节]` | 质量审计 | 派质量优化部对比 |
| `运营 [作品]` | 运营启动 | 派运营部输出运营策略 |
| `分析 [作品]` | 数据分析 | 派数据分析部输出数据报告 |
| `营销 [作品]` | 营销启动 | 派内容营销部输出营销方案 |
| `IP评估 [作品]` | IP评估 | 派IP开发部输出IP评估报告 |
| `继续` | 继续推进 | 检查进度并让对应部门继续 |
| `状态` | 项目汇报 | 汇总各部门进度 |
| `查看锚点` | 查看核心文件 | 从各部门调取当前数据 |

### 专项命令

| 输入 | 相当于 | 执行部门 |
|------|--------|---------|
| `拆解 [书名]` | 竞品分析 | 市场调研部 |
| `优化文档` | 全量优化 | 质量优化部 |
| `扩写` | 字数诊断 | 生产部 |
| `校对 [章节]` | 质量审核 | 编辑部 |
| `运营日报` | 运营数据 | 运营部 |
| `数据分析` | 数据报告 | 数据分析部 |
| `书名优化` | 书名方案 | 内容营销部 |
| `IP评估` | IP价值 | IP开发部 |

## 五、部门间数据流转

```
市场调研部 → 策划部 → 技术部 → 编辑部
    ↓                                    ↓
  调研报告                          人设灵魂卡
                                          ↓
                                    生产部（+ 质量优化部介入）
                                          ↓
                                    编辑部（校对）
                                          ↓
                                    质量优化部（对比）
                                          ↓
                                    生产部（定稿）
                                          ↓
                                    ┌────┴────┐
                                    ↓         ↓
                                运营部    内容营销部
                              （发布）    （推广）
                                    ↓         ↓
                                数据分析部（追踪效果）
                                    ↓
                                IP开发部（完本后）
```

每个流转都是：**前部门出 → CEO传递 → 后部门入**
