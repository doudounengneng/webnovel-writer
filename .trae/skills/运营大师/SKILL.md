## 平台策略

### 主流平台分析维度

| 平台 | 流量特点 | 付费模式 | 适合类型 | 算法偏好 |
|------|---------|---------|---------|---------|
| **起点中文网** | 老牌流量大 | 付费订阅 | 玄幻/都市/历史 | 追读率>收藏率 |
| **番茄小说** | 免费流量大 | 广告分成 | 爽文/快节奏 | 完读率>追读率 |
| **七猫小说** | 下沉市场 | 广告分成 | 都市/赘婿/战神 | 开篇留存率 |
| **飞卢小说** | 同人/创新 | 付费订阅 | 同人/系统流 | 更新频率>质量 |
| **掌阅** | 中腰部 | 付费订阅 | 女频/言情 | 情感线>剧情线 |

### 平台选择策略

```
新书首发策略：
  1. 确定作品类型
  2. 匹配最佳平台（见上表）
  3. 分析该平台当前TOP10同类作品
  4. 确定差异化卖点
  5. 制定首发更新计划

多平台策略：
  主平台：日更（保质量）
  副平台：同步更新（吃流量）
  注意：合同独家条款
```

## 更新节奏管理

### 最佳更新时间

| 时段 | 适合 | 说明 |
|------|------|------|
| 06:00-08:00 | 早起党 | 通勤阅读高峰 |
| 12:00-14:00 | 午休党 | 午休阅读高峰 |
| 18:00-20:00 | 下班党 | 通勤+晚饭后 |
| 22:00-00:00 | 夜猫党 | 睡前阅读高峰（流量最大） |

### 更新频率策略

```
日更策略：
  日更1章（4000-6000字）→ 稳定追读
  日更2章（8000-12000字）→ 冲榜期
  日更3章（12000-18000字）→ 新书首发期（前7天）

加更规则：
  推荐票每500票加1更
  月票每100票加1更
  打赏每100元加1更
  追读每涨10%加1更

断更预警：
  提前3天公告
  补更承诺（断1补2）
  设置存稿箱（至少存5章）
```

## 读者互动体系

### 评论回复策略

```
回复优先级：
  P0：付费读者的评论（30分钟内回复）
  P1：长评/分析贴（2小时内回复）
  P2：普通评论（24小时内回复）
  P3：喷子/恶意评论（忽略或统一回复）

回复技巧：
  1. 感谢型："谢谢支持，后续更精彩"
  2. 互动型："你觉得主角下一步会怎么做？"
  3. 剧透型（适度）："这个伏笔后面会回收"
  4. 求助型："大家想看什么类型的爽点？"
```

### 粉丝群运营

```
群规：
  1. 禁止剧透（除非有剧透专区）
  2. 每日打卡（签到有福利）
  3. 每周投票（决定加更/番外方向）

群活动：
  每周：角色人气投票
  每月：剧情走向投票
  每卷：番外征集
  完本：角色结局投票
```

## 付费设计

### 付费点布局

```
免费章节（前20-30章）：
  目标：吸引读者，建立追读
  策略：每章都有爽点，章末强钩子

VIP章节（第20-30章后）：
  目标：转化付费读者
  策略：付费首章必须有爆点

付费节奏：
  每10章一个小高潮（刺激付费）
  每30章一个大高潮（防止流失）
  每卷结尾留悬念（保证续订）
```

### 定价策略

```
标准定价：
  千字5分（起点标准）
  千字3分（番茄/七猫广告分成）

加更定价：
  正常更新：标准价
  加更章节：标准价（不加价）
  番外/特别篇：可适当提价

促销策略：
  新书上架：前3天打折
  节假日：双倍更新
  完本：全集打折
```
